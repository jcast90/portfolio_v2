Problem:

Before an application is developed, it is important to run tests to make sure that the application’s functions are working properly. The goal was to run specific tests for an RSS Feed Reader application that uses Google's RSS API and have them all pass. Another part of this challenge included picking up the tests where another developer had left off, thus having to decipher their code before proceeding the tests.

Solution:

First, learn the Jasmine testing framework. Then, read and decipher the previous developer’s code and testing framework. Case Study

Problem:

The goal of this project was to create a single-page web application, built using the Knockout framework, that displays a Google Map of an area and various points of interest. The map should be made to reflect your very own neighborhood.

Solution:

Using JavaScript, I was able to have a functioning map with hard-coded locations and a search option. Arcade Game Clone

Case Study

Problem:

After being given half of the HTML5 code for a partially developed game, I needed to create the rest of the game using Object Oriented JavaScript. Having only half of the code meant reading someone else’s code and modifying it to match my JavaScript. Not only that, but I had to add enemies, and have them collide with the player.

Solution

The best solution was to start building the map, then the enemies and their functions and finally the main character and his functions. After going through different processes for how an enemy can affect the player, I ultimately went with a for loop and a math.random(). This allowed for a random number of enemies between a certain number to be generated. Ultimately, I created an HTML5 Canvas powered video game, developed using the best practices in Object Oriented JavaScript.

Website Optimization

Case Study

Problem:

In this project, we were given a website that was slow, and had choppy animation and had a very high load time. The problem, was going into someone else code, understanding what they were doing, then take it apart and work with it so the page would load at an optimal speed and not have choppy frames in their animation.

Solution:

In order to increase the sites load time, the ‘googlefont’ was loaded into the HTML page with javascript, the css was moved to the end of the HTML document. The next step to solving the 60fps for the animation was to go through the javascript. First, I pulled out a function that was creating a new pizza with every for loop. Instead, it was created once, and another function was used to run through the container size and generate pizzas. Next, I limited the amount of pizzas that were being generated within the container. Finally, I went to the variable containing the pizza sizes and optimized them to be only the size that needed to be loaded. Ultimately, I optimized an inefficient web application's JavaScript, CSS and assets delivery, ensuring it runs at 60fps and achieves a PageSpeed score of at least 90.
